Chapter 28: HTML Script & Link Tags

Introduction

HTML alone creates structure. CSS adds styling, and JavaScript adds interactivity. The <link> tag connects CSS files, and the <script> tag connects JavaScript files to your HTML page.

1. The <link> Tag - Connecting CSS

The <link> tag is used to link external CSS files. It goes inside the <head> section and is self-closing.

<!DOCTYPE html>
<html>
<head>
  <title>My Page</title>

  <!-- Link external CSS file -->
  <link rel="stylesheet" href="styles.css">

  <!-- Link multiple CSS files -->
  <link rel="stylesheet" href="reset.css">
  <link rel="stylesheet" href="main.css">
</head>
<body>
  <h1>Styled with external CSS</h1>
</body>
</html>
Important Attributes:
rel="stylesheet" - Defines relationship, must be "stylesheet" for CSS
href="styles.css" - Path to CSS file
media="screen" - Optional, specifies device type

2. The <script> Tag - Connecting JavaScript

The <script> tag is used to embed or link JavaScript. It can go in <head> or before closing </body>.

Method 1: External JS File

<!DOCTYPE html>
<html>
<head>
  <title>My Page</title>
</head>
<body>
  <h1>Hello World</h1>
  <button id="btn">Click Me</button>

  <!-- Link external JS file at end of body -->
  <script src="script.js"></script>
</body>
</html>

Method 2: Inline JavaScript

<script>
  // JavaScript code directly in HTML
  document.getElementById('btn').onclick = function() {
    alert('Button clicked!');
  };
</script>

3. Where to Place Scripts?

Location Pros Cons
In <head> Loads early Blocks HTML parsing, slower page load
Before </body> HTML loads first, faster page JS runs after HTML loads
With defer Downloads early, runs after HTML Best of both worlds
With async Downloads without blocking Execution order not guaranteed

4. Script Attributes: defer vs async

<!-- Defer: Runs after HTML is parsed, maintains order -->
<script src="script1.js" defer></script>
<script src="script2.js" defer></script>

<!-- Async: Runs as soon as downloaded, no order guarantee -->
<script src="analytics.js" async></script>
When to use:
defer - For scripts that need DOM ready, like your main app.js
async - For independent scripts like Google Analytics

5. Complete File Structure Example

project/
│
├── index.html
├── css/
│ ├── reset.css
│ └── style.css
├── js/
│ ├── main.js
│ └── utils.js
└── images/
    └── logo.png
<!-- index.html -->
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>My Website</title>

  <!-- CSS files in head -->
  <link rel="stylesheet" href="css/reset.css">
  <link rel="stylesheet" href="css/style.css">
</head>
<body>
  <header id="header">
    <img src="images/logo.png" alt="Logo">
    <h1>Welcome</h1>
  </header>

  <main>
    <button id="btn">Click Me</button>
  </main>

  <!-- JS files before closing body -->
  <script src="js/utils.js" defer></script>
  <script src="js/main.js" defer></script>
</body>
</html>

6. Linking External Libraries - CDN

You can link CSS/JS from Content Delivery Networks without downloading files.

<!-- Bootstrap CSS from CDN -->
<link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/bootstrap@5.3.0/dist/css/bootstrap.min.css">

<!-- jQuery from CDN -->
<script src="https://code.jquery.com/jquery-3.7.0.min.js"></script>

<!-- Font Awesome Icons -->
<link rel="stylesheet" href="https://cdnjs.cloudflare.com/ajax/libs/font-awesome/6.4.0/css/all.min.css">

7. Internal CSS with <style> Tag

For small projects, you can write CSS directly in HTML:

<head>
  <style>
    body {
      background: #1e293b;
      color: white;
    }
   .btn {
      padding: 10px 20px;
      background: #0ea5e9;
    }
  </style>
</head>

8. Loading Order - Critical Concept

Browser loads in this order:
1. HTML from top to bottom
2. CSS in <head> - blocks rendering until loaded
3. JS without defer/async - blocks HTML parsing
4. JS with defer - runs after HTML parsed
5. JS with async - runs whenever ready

Best Practice: CSS in head, JS before </body> with defer

9. Common Mistakes ❌

<!-- Wrong: Missing rel attribute -->
<link href="style.css">

<!-- Correct: -->
<link rel="stylesheet" href="style.css">

<!-- Wrong: Script in head without defer -->
<head>
  <script src="app.js"></script> <!-- Blocks page load -->
</head>

<!-- Correct: Use defer or place before /body -->
<script src="app.js" defer></script>

<!-- Wrong: Self-closing script tag -->
<script src="app.js" /> <!-- Won't work -->

<!-- Correct: Closing tag required -->
<script src="app.js"></script>

10. Favicon with Link Tag

<head>
  <!-- Favicon -->
  <link rel="icon" type="image/png" href="favicon.png">

  <!-- Apple Touch Icon -->
  <link rel="apple-touch-icon" href="apple-icon.png">
</head>
✅ Best Practices Summary:
1. Put <link rel="stylesheet"> in <head>
2. Put <script> before </body> with defer
3. Use external files for better caching and maintainability
4. Use defer for your scripts, async for third-party
5. Load CSS before JS - prevents Flash of Unstyled Content
6. Use CDNs for popular libraries - faster loading
7. Minify CSS/JS files in production
