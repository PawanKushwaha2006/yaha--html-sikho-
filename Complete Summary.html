Chapter OneWhat is HTML?

A full introduction to HTML — why it was created, how it works, and why it matters.

🔑 Definition

HTML = HyperText Markup Language

  • HyperText — Text that contains links connecting to other pages.
  • Markup — Describing and structuring content using tags.
  • Language — The language that browsers understand.
What does HTML do?

HTML creates the structure (skeleton) of a webpage. Just like a building has walls and beams, HTML is the foundation of every webpage. It tells the browser where the heading is, where the paragraph is, and where the image goes.

HTML doesn't work alone
  • 🏗️ HTML — Structure (the bones)
  • 🎨 CSS — Design (the appearance)
  • ⚡ JavaScript — Behaviour (the interactivity)
HTML is NOT a programming language — it is a Markup Language. It has no logic, loops, or conditions. Its only job is to give structure and meaning to content.
How does a browser read HTML?

When you type a URL in a browser, the server sends back an HTML file. The browser parses (reads) that HTML, builds a DOM (Document Object Model), and renders it visually on your screen as a webpage.

Chapter TwoHistory of HTML

When HTML was born, who created it, and how many versions have come so far.

1989–1991
Tim Berners-Lee invented HTML at CERN (Switzerland). The World Wide Web was born — originally only for scientists to share research documents.
1993 — HTML 1.0
The first official version. Very few tags — only headings, paragraphs, links, and images. Extremely basic by today's standards.
1995 — HTML 2.0
Forms arrived! Users could submit data for the first time. Tables were also introduced, enabling structured data display.
1997 — HTML 3.2 & 4.0
CSS support was added. JavaScript integration happened. The web began to be used professionally and commercially.
1999 — HTML 4.01
A very stable, long-lasting version. Officially approved by W3C. Remained the standard for many years.
2000–2008 — XHTML
An XML-strict version of HTML. Every tag had to be properly closed. Became less popular over time due to its rigid rules.
2014 — HTML5 HTML5
The biggest update ever! Native Video & Audio support, semantic tags (header, nav, footer), Canvas, Local Storage, Geolocation API — everything was new and powerful.
Today — HTML Living Standard
WHATWG (Apple, Mozilla, Google, Microsoft) now collaboratively maintain HTML as a continuously updated "Living Standard" — no more version numbers.

Chapter ThreeDocument Structure

Every HTML page has a basic skeleton — this is the foundation everything is built upon.

What is a Tag?

HTML Tags are written inside angle brackets < >. Most tags come in pairs — an opening tag <p> and a closing tag </p>. Content is written between them. Some tags are self-closing, like <br>, <img>, and <hr>.

index.html — Complete Boilerplate
<!-- Step 1: DOCTYPE declaration -->
<!DOCTYPE html>

<!-- Step 2: Root element with language attribute -->
<html lang="en">

<!-- Step 3: HEAD section (info for browser, not visible) -->
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>My First Page</title>
  <link rel="stylesheet" href="style.css">
</head>

<!-- Step 4: BODY section (everything visible to the user) -->
<body>
  <h1>Hello, World!</h1>
  <p>This is my first HTML page.</p>
</body>

</html>
<!DOCTYPE html>
Tells the browser this is an HTML5 document. Always write this first — without it, the browser enters "quirks mode".
<html>
The root element — wraps the entire page. Use the lang attribute to declare the page language.
<head>
Contains meta information for the browser and search engines. Nothing inside <head> is displayed on screen.
<title>
Sets the page name shown in the browser tab and in Google search results.
<body>
Everything written here is visible to the user on screen — text, images, forms, videos, etc.
<meta>
Provides extra page info — charset encoding, viewport settings, description, and keywords for SEO.

Chapter SevenLinks — The <a> Tag

The most important tag in HTML — hyperlinks are what make the internet an internet.

All Uses of the <a> Tag
<!-- 1. External website (opens in new tab) -->
<a href="https://google.com" target="_blank" rel="noopener noreferrer">Google</a>

<!-- 2. Internal page on the same website -->
<a href="about.html">About Us</a>

<!-- 3. Jump to a section on the same page (anchor link) -->
<a href="#section2">Go to Section 2</a>
<div id="section2">This is where it lands</div>

<!-- 4. Email link (opens mail app) -->
<a href="mailto:info@example.com?subject=Hello">Send Email</a>

<!-- 5. Phone link (dials on mobile) -->
<a href="tel:+919876543210">Call Us</a>

<!-- 6. WhatsApp link -->
<a href="https://wa.me/919876543210">WhatsApp</a>

<!-- 7. Download a file -->
<a href="brochure.pdf" download="my-brochure.pdf">Download PDF</a>

<!-- 8. Image as a clickable link -->
<a href="home.html">
  <img src="logo.png" alt="Logo">
</a>

Chapter EightImages

All the ways to display images on a webpage — from basic to fully responsive.

<img> — Basic to Advanced
<!-- Basic image -->
<img src="photo.jpg" alt="A mountain landscape">

<!-- Fixed width and height -->
<img src="photo.jpg" alt="..."
     width="400" height="300">

<!-- Lazy loading for performance -->
<img src="photo.jpg" alt="..."
     loading="lazy">

<!-- Responsive image (different sizes for different screens) -->
<img
  src="medium.jpg"
  srcset="small.jpg 480w,
          medium.jpg 800w,
          large.jpg 1200w"
  sizes="(max-width: 600px) 480px, 800px"
  alt="Responsive image">

<!-- Figure with caption -->
<figure>
  <img src="taj.jpg" alt="Taj Mahal">
  <figcaption>The Taj Mahal, Agra, India</figcaption>
</figure>

<!-- Picture element (serve different formats) -->
<picture>
  <source srcset="img.webp" type="image/webp">
  <source srcset="mobile.jpg" media="(max-width: 600px)">
  <img src="fallback.jpg" alt="Description">
</picture>
⚠️ Why is alt attribute required?
  • If the image fails to load, the alt text is displayed instead.
  • Screen readers read alt text aloud for visually impaired users.
  • Google reads alt text to understand and rank images in search.
  • Never leave it empty unless the image is purely decorative.
✅ Image Formats Guide
  • JPG/JPEG — Best for photos, small file size
  • PNG — Use when you need transparent backgrounds
  • WebP — Modern format, smaller than JPG/PNG
  • SVG — Vector icons/logos, infinitely scalable
  • GIF — Simple looping animations
  • AVIF — Newest format, even smaller than WebP

Chapter NineTables

Organizing data into rows and columns using HTML tables.

Complete Table Structure
<table>

  <!-- Optional: title for the table -->
  <caption>Student Results 2024</caption>

  <!-- Control column widths -->
  <colgroup>
    <col style="width: 40%">
    <col span="2" style="width: 30%">
  </colgroup>

  <!-- Table Header -->
  <thead>
    <tr>
      <th scope="col">Name</th>
      <th scope="col">Maths</th>
      <th scope="col">Science</th>
    </tr>
  </thead>

  <!-- Table Body -->
  <tbody>
    <tr>
      <td>Rahul</td>
      <td>95</td>
      <td>88</td>
    </tr>
    <tr>
      <!-- colspan: merge 2 columns horizontally -->
      <td colspan="2">Priya (absent)</td>
      <td>—</td>
    </tr>
    <tr>
      <td>Arjun</td>
      <!-- rowspan: merge 2 rows vertically -->
      <td rowspan="2">Combined</td>
      <td>91</td>
    </tr>
  </tbody>

  <!-- Table Footer (totals, averages) -->
  <tfoot>
    <tr>
      <td>Average</td>
      <td>91.5</td>
      <td>89.5</td>
    </tr>
  </tfoot>

</table>
colspan and rowspan — Merging Cells
  • colspan="2" — One cell takes up 2 columns (horizontal merge)
  • rowspan="3" — One cell takes up 3 rows (vertical merge)
  • When you merge cells, remove the cells that are being "covered" by the merge.
⚠️ Tables are NOT for Layout

Never use <table> to create page layouts (like side-by-side columns). Tables are only for tabular/grid data. Use CSS Flexbox or Grid for layouts instead.

Chapter ElevenSemantic HTML5

Meaningful tags that describe the purpose of content — replacing endless <div> soup with real structure.

🤔 Why does Semantic HTML matter?
  • ✅ SEO — Google understands your page structure and ranks it better.
  • ✅ Accessibility — Screen readers give blind users proper navigation through the page.
  • ✅ Readability — Code is much easier to read and maintain.
  • ✅ Collaboration — Other developers instantly understand the page structure.
  • ❌ Avoid "div soup" — stuffing everything into nested <div> tags with no meaning.
Full Semantic Page Layout
<body>

  <header>             <!-- Logo, navigation, page title -->
    <nav>             <!-- Navigation menu links -->
      <a href="/">Home</a>
      <a href="/about">About</a>
    </nav>
  </header>

  <main>               <!-- Primary page content (use only once) -->

    <section>         <!-- Thematic content group -->
      <h2>Latest Articles</h2>

      <article>       <!-- Self-contained piece (blog post, news item) -->
        <h3>Breaking News</h3>
        <p>Content here...</p>
        <time datetime="2024-05-04">4 May 2024</time>
      </article>

    </section>

    <aside>           <!-- Sidebar, related articles, ads -->
      <p>Related links...</p>
    </aside>

    <!-- Expandable FAQ accordion -->
    <details>
      <summary>What is HTML5?</summary>
      <p>HTML5 is the latest version of HTML...</p>
    </details>

    <!-- Progress bar -->
    <progress value="70" max="100"></progress>

    <!-- Meter (gauge) -->
    <meter min="0" max="10" low="3" high="7" value="6"></meter>

  </main>

  <footer>             <!-- Copyright, contact info, links -->
    <address>
      Contact: <a href="mailto:hi@site.com">hi@site.com</a>
    </address>
  </footer>

</body>
<header>
Top of a page or section — contains logo, site name, navigation
<nav>
A group of navigation links — main menu, breadcrumbs, pagination
<main>
The primary, dominant content of the page — use only once per page
<section>
A thematic grouping of related content, usually with its own heading
<article>
Self-contained, independently reusable content — blog post, news article, comment
<aside>
Tangentially related content — sidebar, pull quote, ads, related links
<footer>
Bottom of a page or section — copyright, contact, social links
<details> + <summary>
A click-to-expand disclosure widget — great for FAQs and accordions
<dialog>
A modal dialog box or popup window — native, no JavaScript needed to show
<figure> + <figcaption>
A self-contained figure (image, diagram, code) with an optional caption
<time>
A human-readable time/date with a machine-readable datetime attribute
<time datetime="2024-01-15">Jan 15</time>
<progress>
A visual progress bar — file upload, loading status
<progress value="70" max="100">
<meter>
A scalar measurement within a known range — fuel gauge, disk usage, ratings
<meter min="0" max="10" value="7">
<canvas>
A drawing surface — used with JavaScript for games, charts, animations
<svg>
Inline vector graphics — scalable icons, logos, diagrams directly in HTML
<template>
Holds client-side HTML that is not rendered — cloned and used via JavaScript
<output>
Displays the result of a calculation — useful inside forms
<mark>
Highlighted/relevant text — like a yellow highlighter pen on text

Chapter SixteenBest Practices & Tips

The rules and habits that separate professional HTML from beginner HTML.

✅ Always Do This
  • Put <!DOCTYPE html> on the very first line of every page.
  • Add <html lang="en"> — always declare the language.
  • Every <img> must have a meaningful alt attribute.
  • Use only one <h1> per page — crucial for SEO.
  • Always link <label> to its <input> using for and id.
  • Use semantic elements (<header>, <nav>, <main>, etc.).
  • Keep CSS in a separate style.css file.
  • Load JavaScript at the end of body or use the defer attribute.
  • Indent your code consistently for readability.
  • Always include the viewport meta tag for mobile responsiveness.
  • Always specify width and height on images to prevent layout shifts.
❌ Never Do This
  • Don't use inline styles everywhere — it's impossible to maintain.
  • Never use <table> for page layout — use CSS Grid or Flexbox.
  • Don't spam <br> tags for spacing — use CSS margins.
  • Avoid "div soup" — nesting dozens of <div>s with no semantic meaning.
  • Don't confuse <b> (visual) with <strong> (semantic).
  • Never use deprecated tags — <font>, <center>, <marquee>, <blink>.
  • Don't leave alt="" unless the image is truly decorative.
  • Avoid href="javascript:void(0)" — use proper buttons or anchors.
  • Don't skip heading levels — h1 → h3 (skipping h2) is bad practice.
🚀 Performance Tips
  • <img loading="lazy"> — Images below the fold load only when scrolled to.
  • <script defer> — JS is downloaded in parallel but executed after HTML parsing.
  • Use WebP image format — significantly smaller than JPG or PNG.
  • Always specify image width and height to prevent Cumulative Layout Shift (CLS).
  • Use <link rel="preload"> for critical fonts and resources.
  • Minify your HTML in production — remove unnecessary whitespace and comments.
HTML is easy to learn — but writing it correctly is an art. Writing semantic, accessible, and performance-friendly HTML is the true mark of a professional web developer.
