Chapter 22: HTML Entities

What are HTML Entities?

HTML Entities are special codes used to display reserved characters, symbols, and characters that don't exist on your keyboard. Some characters like < and > have special meaning in HTML, so we use entities to display them.

1. Why Do We Need Entities?

  • Reserved Characters: <, >, & are used in HTML syntax
  • Invisible Characters: Non-breaking space, em dash
  • Special Symbols: ©, ®, ™, €, £, ¥
  • Math Symbols: ±, ×, ÷, ≤, ≥, ∞
  • Emoji-like: ♥, ★, ☺, →

2. Entity Syntax

Entities start with & and end with ;

&entity_name;
OR
&#entity_number;
Example: To display < write &lt; or &#60;
Both will show: <

9. Non-Breaking Space - &nbsp;

Regular spaces collapse in HTML. &nbsp; creates space that won't break into new line.

<p>Hello&nbsp;&nbsp;&nbsp;World</p>
<p>100&nbsp;km/h</p>

Hello   World

100 km/h

10. Displaying HTML Code

To show HTML tags as text, use entities:

<p>To create a paragraph use &lt;p&gt; tag</p>
<p>The &lt;div&gt; tag is a container</p>

To create a paragraph use <p> tag

The <div> tag is a container

✅ Best Practices:
1. Always use entities for <, >, & in content
2. Use &nbsp; to prevent unwanted line breaks
3. Use entity names when possible - easier to read than numbers
4. For copyright footer: &copy; 2026 Company Name
5. Test special characters across browsers
6. Use UTF-8 charset to support most symbols directly

11. Common Mistakes ❌

<!-- Wrong: Using < directly -->
<p>5 < 10</p> <!-- Browser may think < is tag start -->

<!-- Correct: Use entity -->
<p>5 &lt; 10</p>

<!-- Wrong: Missing semicolon -->
<p>&copy 2026</p>

<!-- Correct: -->
<p>&copy; 2026</p>
