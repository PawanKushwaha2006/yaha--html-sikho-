HTML Comments
1. What are HTML Comments?

Comments are notes in your code that browsers ignore. Users cannot see them on the web page.

They are only visible when someone opens the source code with "View Page Source".

Syntax: <!-- Comment Here -->

Starts with: <!--

Ends with: -->

Everything between these is a comment.

Note: Comments help you and other developers understand the code. They don't affect page speed or layout.
2. Single Line Comment

Example:

<!-- This is a single line comment -->
<p>This paragraph is visible</p>
OUTPUT ON WEB PAGE:

This paragraph is visible

The comment above the paragraph is completely hidden.

3. Multi-Line Comment

You can write comments across multiple lines. Everything between <!-- and --> is ignored.

Example:

<!--
  This is a multi-line comment.
  You can write anything here.
  Browser will ignore all these lines.
-->
<h1>Website Title</h1>
OUTPUT ON WEB PAGE:

Website Title

4. Why Use Comments? - 5 Main Uses

1. Explain Complex Code

<!-- This formula calculates total price with 18% GST -->
<p>Total: $118</p>

2. Mark Sections of Code

<!-- Header Section Starts -->
<header>...</header>
<!-- Header Section Ends -->

3. Temporarily Disable Code

<!-- <p>Old text - not using now</p> -->
<p>New updated text</p>

4. Add TODO Notes

<!-- TODO: Add contact form here later -->
<!-- FIXME: Mobile layout broken -->

5. Author & Date Info

<!-- Author: John Doe -->
<!-- Date: 28 April 2026 -->
<!-- Version: 2.0 -->
5. Commenting Out Code - Debugging Trick

If some code is causing errors, you can "comment it out" instead of deleting. This is called debugging.

Before Commenting:

<p>This will show</p>
<p>This is buggy code</p>
<p>This will also show</p>

After Commenting Out:

<p>This will show</p>
<!-- <p>This is buggy code</p> -->
<p>This will also show</p>
OUTPUT:

This will show

This will also show

✓ Pro Tip: Comment out code instead of deleting. If you need it later, just remove <!-- and -->
6. Common Mistakes with Comments
MistakeWrong CodeProblem
Space in syntax <! -- Comment -- > Space after ! breaks it
Nested comments <!-- <!-- Nested --> --> Comments inside comments don't work
Forgetting to close <!-- Comment Rest of page becomes comment
Using // or /* */ // HTML Comment That is for CSS/JS, not HTML
✗ Warning: Never put passwords, API keys, or sensitive data in comments. Anyone can see source code with Ctrl+U.
7. Comments vs Other Languages

Different languages use different comment syntax:

LanguageSingle LineMulti Line
HTML<!-- Comment --><!-- Multi -->
CSS/* Comment *//* Multi */
JavaScript// Comment/* Multi */
Python# Comment""" Multi """
Remember: In HTML files, always use <!-- --> even if you write CSS or JS inside. But inside <style> use /* */ and inside <script> use //
