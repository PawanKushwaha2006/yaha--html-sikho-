Chapter 26: HTML Classes & IDs

What are Classes and IDs?

Classes and IDs are "hooks" or "labels" you add to HTML elements. CSS uses them to style elements, and JavaScript uses them to select and manipulate elements.

Feature class id
Purpose Group multiple elements Uniquely identify one element
Syntax class="name" id="name"
CSS Selector .name #name
JS Selector document.querySelectorAll('.name') document.getElementById('name')
Reusability Can use on many elements Must be unique per page
Multiple Values Yes: class="btn primary" No: Only one id allowed

1. The class Attribute

Use classes when multiple elements need the same styling or behavior.

<!-- HTML -->
<div class="card">Card 1</div>
<div class="card">Card 2</div>
<div class="card">Card 3</div>

<p>This is <span class="highlight">important</span> text.</p>
<p>This is <span class="error">error</span> message.</p>
/* CSS - targets all elements with class="card" */
.card {
  background: rgba(14, 165, 233, 0.1);
  border: 2px solid #0ea5e9;
  padding: 20px;
  border-radius: 8px;
}

.highlight {
  background: #f59e0b;
  color: #0f172a;
  padding: 3px 8px;
}

Output:

Card 1 - All cards share same style
Card 2 - Same class = same CSS

This is important text.

This is error message.

2. Multiple Classes

An element can have multiple classes separated by spaces.

<button class="btn btn-primary btn-large">Submit</button>
<p class="text-center error">Centered error text</p>
/* Each class adds styles */
.btn { padding: 10px 20px; border: none; }
.btn-primary { background: #0ea5e9; color: white; }
.btn-large { font-size: 18px; }
.text-center { text-align: center; }
.error { color: #ef4444; }

3. The id Attribute

Use IDs when you need to uniquely identify ONE specific element. IDs must be unique per page.

<!-- HTML -->
<h1 id="main-title">Welcome to My Site</h1>
<button id="demo-btn">Click Me</button>
<div id="header">Header Section</div>
/* CSS - # targets the id */
#main-title {
  color: #a855f7;
  font-size: 2rem;
  text-align: center;
}

#demo-btn {
  background: #10b981;
  color: white;
}

Output:

Welcome to My Site

4. JavaScript with Classes & IDs

<!-- HTML -->
<p id="message">Original text</p>
<button id="change-btn">Change Text</button>

<div class="box">Box 1</div>
<div class="box">Box 2</div>
<div class="box">Box 3</div>
// JavaScript - Select by ID
document.getElementById('change-btn').onclick = function() {
  document.getElementById('message').textContent = 'Text Changed!';
};

// JavaScript - Select by Class
let boxes = document.querySelectorAll('.box');
boxes.forEach(box => {
  box.style.background = '#0ea5e9';
});

Original text

5. Naming Rules

  • Start with letter: class="btn" ✅, class="1btn" ❌
  • Use hyphens: class="main-header" ✅, class="main header" ❌
  • Case sensitive: class="Header" ≠ class="header"
  • No spaces: class="btn-primary" ✅, class="btn primary" = 2 classes
  • Be descriptive: class="nav-link" ✅, class="x" ❌

6. When to Use Class vs ID

Use class when:
1. Styling multiple similar elements - cards, buttons, links
2. Grouping elements for JavaScript - all .tab elements
3. Reusable styles - .text-center, .btn-primary

Use id when:
1. Unique element - #header, #main-nav, #footer
2. JavaScript needs one specific element - #submit-btn
3. Anchor links - <a href="#section1">
4. Form labels - <label for="email"> matches id="email"

8. Common Mistakes ❌

<!-- Wrong: Duplicate IDs -->
<div id="header">Header 1</div>
<div id="header">Header 2</div> <!-- IDs must be unique -->

<!-- Correct: Use class instead -->
<div class="header">Header 1</div>
<div class="header">Header 2</div>

<!-- Wrong: Space in class name -->
<div class="main header">...</div> <!-- This is 2 classes -->

<!-- Correct: Use hyphen -->
<div class="main-header">...</div>

<!-- Wrong: Starting with number -->
<div class="1st-item">...</div>

<!-- Correct: Start with letter -->
<div class="first-item">...</div>

9. CSS Specificity

IDs have higher specificity than classes. #id beats .class

/* This wins */
#title { color: red; }

/* This loses */
.title { color: blue; }

<h1 id="title" class="title">Text</h1> <!-- Will be red -->
✅ Quick Summary:
Class =.name - Reusable, for groups, multiple per element
ID = #name - Unique, for one element only, higher priority
Rule of thumb: Use classes for styling, IDs for JavaScript hooks and unique sections
