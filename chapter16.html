Chapter 16: Selection Menus

Learn to create dropdowns with <select> and searchable inputs with <datalist>. Perfect for country pickers, categories, and autocomplete fields.

1. The <select> Element - Dropdowns

Creates a dropdown menu. Users pick ONE option from a list. Each option uses <option> tag.

Basic Syntax:

<label for="country">Select Country:</label>
<select id="country" name="country">
  <option value="us">United States</option>
  <option value="uk">United Kingdom</option>
  <option value="in">India</option>
</select>
Output:
Key Points:
1. <select> = Container for dropdown
2. <option> = Each choice in the list
3. value attribute = What gets sent to server
4. Text between <option> tags = What user sees

2. <option> Attributes

AttributePurposeExample
valueData sent to server<option value="html">HTML</option>
selectedDefault selected option<option selected>CSS</option>
disabledCannot be selected<option disabled>Coming Soon</option>
labelShort label for option<option label="JS">JavaScript</option>

Example: Default & Disabled Options

<select name="skill">
  <option value="" disabled selected>-- Choose a skill --</option>
  <option value="html">HTML</option>
  <option value="css">CSS</option>
  <option value="js" disabled>JavaScript (Coming Soon)</option>
</select>

3. <optgroup> - Group Options

Group related options under a heading. Creates visual sections in dropdown.

<select name="tech">
  <optgroup label="Frontend">
    <option value="html">HTML</option>
    <option value="css">CSS</option>
    <option value="js">JavaScript</option>
  </optgroup>
  <optgroup label="Backend">
    <option value="node">Node.js</option>
    <option value="python">Python</option>
    <option value="php">PHP</option>
  </optgroup>
</select>
Output:

4. Multiple Selection

Add multiple attribute to allow selecting multiple options. Hold Ctrl/Cmd to select more.

<label for="skills">Select Skills (Hold Ctrl):</label>
<select id="skills" name="skills" multiple size="5">
  <option value="html">HTML</option>
  <option value="css">CSS</option>
  <option value="js">JavaScript</option>
  <option value="react">React</option>
  <option value="node">Node.js</option>
</select>
Output - Hold Ctrl/Cmd to select multiple:
Attributes:
multiple = Allow multiple selections
size="5" = Show 5 options at once instead of dropdown

5. The <datalist> Element - Autocomplete

Provides a list of predefined options for an <input>. User can type or pick from list. Best of both worlds: free text + suggestions.

Basic Syntax:

<label for="browser">Choose a browser:</label>
<input list="browsers" id="browser" name="browser">

<datalist id="browsers">
  <option value="Chrome">
  <option value="Firefox">
  <option value="Safari">
  <option value="Edge">
</datalist>
Output - Type or select:
Key Difference:
<select> = User MUST pick from list
<datalist> = User CAN pick OR type custom value

Example: Country with Search

<input list="countries" name="country" placeholder="Start typing...">
<datalist id="countries">
  <option value="United States">
  <option value="United Kingdom">
  <option value="India">
  <option value="Canada">
  <option value="Australia">
</datalist>

6. <select> vs <datalist> - When to Use?

Feature<select><datalist>
User InputOnly from listFrom list OR custom
Use CaseFixed choices: Gender, CountrySearchable: Tags, Cities, Skills
ValidationAlways validCan be invalid if user types
UXDropdown clickType to filter + dropdown
Best ForSmall lists <20 itemsLarge lists 100+ items

7. Complete Real World Example

<form action="/submit" method="post">
  <h3>Job Application Form</h3>

  <!-- Select Dropdown -->
  <label for="position">Position Applied For:</label>
  <select id="position" name="position" required>
    <option value="" disabled selected>-- Select Position --</option>
    <optgroup label="Engineering">
      <option value="frontend">Frontend Developer</option>
      <option value="backend">Backend Developer</option>
      <option value="fullstack">Full Stack Developer</option>
    </optgroup>
    <optgroup label="Design">
      <option value="ui">UI Designer</option>
      <option value="ux">UX Designer</option>
    </optgroup>
  </select>

  <!-- Multiple Select -->
  <label for="skills">Skills (Hold Ctrl to select multiple):</label>
  <select id="skills" name="skills" multiple size="4">
    <option value="html">HTML</option>
    <option value="css">CSS</option>
    <option value="js">JavaScript</option>
    <option value="react">React</option>
    <option value="node">Node.js</option>
  </select>

  <!-- Datalist -->
  <label for="location">Preferred Location:</label>
  <input list="cities" id="location" name="location" placeholder="Type or select city">
  <datalist id="cities">
    <option value="New York">
    <option value="London">
    <option value="Mumbai">
    <option value="Remote">
  </datalist>

  <button type="submit">Apply Now</button>
</form>

9. Styling Tips

/* Custom dropdown arrow */
select {
  appearance: none;
  background-image: url('arrow.svg');
  background-repeat: no-repeat;
  background-position: right 10px center;
  padding-right: 35px;
}

/* Style optgroup */
optgroup {
  font-weight: bold;
  color: #38bdf8;
}

/* Style options */
option {
  padding: 8px;
  background: #1e293b;
}

option:checked {
  background: #0ea5e9;
  color: #000;
}
Browser Limitation: You cannot fully style <option> elements. For custom dropdowns, use libraries like Select2, Choices.js, or build with divs + JS.

10. Summary - Golden Rules

  1. <select>: Use when user must pick from predefined list. Good for small lists
  2. <datalist>: Use when user can pick OR type custom value. Good for large searchable lists
  3. <option>: Always add value attribute for form submission
  4. <optgroup>: Group related options with labels for better UX
  5. multiple: Add to <select> to allow multiple selections
  6. selected: Set default option with <option selected>
  7. disabled: Use for placeholders: <option disabled selected>Choose...</option>
  8. Accessibility: Always pair with <label for="id">
Pro Tip: For mobile apps, <select> opens native picker which is better UX than custom dropdowns. Don't reinvent unless needed!
