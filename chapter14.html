Chapter 15: Form Input Types

HTML5 introduced specialized input types. They provide built-in validation, mobile keyboards, and better UX. We'll cover: email, number, range, date.

1. Input Type: email

For email addresses. Browser validates format and shows email keyboard on mobile.

Basic Syntax:

<label for="email">Email:</label>
<input type="email" id="email" name="email" required>
Live Example:

Key Attributes for Email:

AttributePurposeExample
requiredField must be filled<input type="email" required>
placeholderHint textplaceholder="name@site.com"
multipleAllow multiple emails<input type="email" multiple>
patternCustom regex validationpattern="[a-z0-9._%+-]+@[a-z0-9.-]+\.[a-z]{2,}$"
Validation: Browser checks for @ symbol and domain. But it's basic. For strict validation, use pattern or JavaScript.

Multiple Emails Example:

<input type="email" name="emails" multiple placeholder="Separate with commas">

User can enter: john@mail.com, jane@mail.com

2. Input Type: number

For numeric input only. Shows number keypad on mobile. Has up/down spinners on desktop.

Basic Syntax:

<label for="age">Age:</label>
<input type="number" id="age" name="age">
Live Example:

Key Attributes for Number:

AttributePurposeExample
minMinimum value allowedmin="0"
maxMaximum value allowedmax="100"
stepIncrement/decrement stepstep="0.5" allows 1, 1.5, 2
valueDefault valuevalue="25"

Example: Price Input

<label for="price">Price ($):</label>
<input type="number" id="price" name="price" 
       min="0" max="9999.99" step="0.01" value="99.99">
Output - Accepts decimals:
Warning: type="number" is not for phone numbers, zip codes, or IDs. Those may have leading zeros. Use type="text" with pattern instead.

3. Input Type: range

Creates a slider. User picks a value by dragging. Good for volume, brightness, price filters.

Basic Syntax:

<label for="volume">Volume:</label>
<input type="range" id="volume" name="volume" min="0" max="100" value="50">
Live Example:

Key Attributes for Range:

AttributeDefaultPurpose
min0Minimum value
max100Maximum value
step1Increment value
valuemin + (max-min)/2Default position

Example: Price Filter with Step

<label for="price">Max Price: $<span id="priceVal">500</span></label>
<input type="range" id="price" name="price" 
       min="0" max="1000" step="50" value="500"
       oninput="priceVal.textContent = this.value">
Output - Jumps in steps of 50:
Important: Range doesn't show the value by default. You need JavaScript to display current value like shown above.

Basic Syntax:

<label for="birthday">Birthday:</label>
<input type="date" id="birthday" name="birthday">

maxLatest date allowedmax="2026-12-31"
valueDefault datevalue="2025-05-01"

Example: Booking Date with Limits

<label for="booking">Booking Date:</label>

MistakeWrongCorrect
Using text for email<input type="text"><input type="email"> for validation
No min/max for number<input type="number">Add min="0" max="100" to prevent invalid values
Range without value displayJust <input type="range">Add JS to show current value to user
Wrong date formatvalue="01-05-2026"Must be value="2026-05-01" YYYY-MM-DD
Using number for phone<input type="number">Use <input type="tel"> for phones

7. Browser Compatibility

Input TypeChromeFirefoxSafariMobile
email✅ Full✅ Full✅ Full✅ Email keyboard
number✅ Full✅ Full✅ Full✅ Number keyboard
range✅ Full✅ Full✅ Full✅ Touch slider
date✅ Full✅ Full⚠️ Partial✅ Native picker

8. Summary - Golden Rules

  1. email: Use for emails. Gives validation + mobile keyboard. Add multiple for comma-separated emails
  2. number: Use for quantities, age, price. Always set min and max. Don't use for phone/zip codes
  3. range: Use for values in a range like volume, filters. Always show value with JS. Set step for increments
  4. date: Use for dates. Value format must be YYYY-MM-DD. Set min/max to limit range
  5. Validation: These types give basic browser validation. Add required for mandatory fields
  6. UX: Correct input type = correct mobile keyboard = better user experience
  7. Fallback: Old browsers treat unknown types as type="text". Always add labels
Pro Tip: Always use <label> with for attribute. Clicking label focuses the input. Great for accessibility!
